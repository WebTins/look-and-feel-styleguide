# Beschrijving

In het team is het belangrijk om regels en afspraken te maken. Daarom is het ook handig om regels te majen voor onze werkwijze tijdens het coderen. In de README staat hoe we onze class namen geven en wat onze code conventies zijn.

## Class names

Wanneer we classes gebruiken waar meerdere woorden in zitten, moeten we "-" gebruiken tussen de woorden in. Ook is het belangrijk om geen hoofdletters te gebruiken bij de classes, omdat we hoofdletters gebruiken bij variabelen.

Bijvoorbeeld: $\color{#00FF00}{[class = "artikel-box"]}$

~~$\color{#FF0000}{[class = "artikelbox"]}$~~

~~$\color{#FF0000}{[class = "ArtikelBox"]}$~~

~~$\color{#FF0000}{[class = "artikelBox"]}$~~

~~$\color{#FF0000}{[class = "Arikel-Box"]}$~~


## Code coventies

Schone en onderhoudbare code is erg belangrijk zodat iedereen het kan lezen en begrijpen. Het is dus van belang dat er code conventies zijn waar het team zich aan moet houden. De onderdelen van de Code coventies zijn: Volgorde, Ademruimte en Nesting.

- Volgorde
> Bij Volgorde is het belangrijk om de HTML landmarks, Headings CSS selectors op volgorde te zetten zodat je duidelijk van boven naar beneden kan lezen en begrijpen wat er staat.

- Ademruimte
> Ademruimte is zeer belangrijk voor duidelijk leesbare code, omdat je dan goed kan lezen welke elementen of selectors  
in elkaar zitten. Daarom is het goed om witregels te gaan gebruiken na een sluitings tag.

- Nesting
> Bij nesting is het belangrijk om je media queries apart te gaan gebruiken bij elke class met de selectors, omdat je dan een beter zicht krijgt bij elke class/selector waar de media queries zich in bevinden.
